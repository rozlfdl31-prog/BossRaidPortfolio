# 🛡️ 플레이어 피격 및 사망 로직의 디테일 개선 (Combat Polish)

핵심 메커니즘이 작동하더라도, 플레이어가 느끼는 감각이 부자연스럽다면 몰입도는 떨어진다. 금일 `Boss Raid Portfolio` 프로젝트에서 플레이어의 피격(Hit)과 사망(Death) 과정에서 발견된 논리적 허점을 보완한 과정을 기술한다.

## 1. 공중 피격 시 발생하던 부유 버그 수정

### 1.1. Problem: 중력을 잊은 캡슐
기존의 `HitState`는 단순히 피격 애니메이션을 재생하고 무적 시간을 처리하는 기능만 수행했다. 이로 인해 플레이어가 점프 중이거나 공중에 떠 있는 상태에서 보스에게 피격당하면, 피격 상태가 종료될 때까지 공중에 정지된 채 떠 있는 현상이 발생했다.

### 1.2. Solution: 상태 내 중력 연산 유지
피격 상태에서도 물리 엔진의 기본 법칙이 적용되도록 `HitState` 내 `Update` 루프에 중력 계산 로직을 추가했다.

```csharp
public override void Update()
{
    // 피격 중에도 중력을 적용하여 자연스러운 낙하 유도
    Controller.ApplyGravity();
    Controller.Move(Vector3.zero); // 중력에 의한 수직 이동만 수행

    // 피격 타이머 체크 및 복귀 로직...
}
```
이제 플레이어는 공중에서 피격당하더라도 바닥으로 떨어지며 피격 모션을 취하게 되어, 훨씬 자연스러운 물리 피드백을 제공한다.

## 2. 사망 이후의 좀비 코루틴 정리 (Coroutine Cleanup)

### 2.1. Problem: 죽어도 죽지 않는 무적 시간
플레이어가 체력이 0이 되어 사망하더라도, 피격 직후 시작된 '무적 시간 코루틴'은 백그라운드에서 계속 동작하고 있었다. 이는 사망 후 리스폰 로직이나 게임 오버 연출 시 의도치 않은 상태 변경(`IsInvincible = false`)을 일으켜 논리적 충돌을 유발할 수 있다.

### 2.2. Solution: 상태 전이 시의 강제 정리
사망 로직이 실행되는 즉시 해당 객체에서 실행 중인 모든 비동기 작업을 중단하도록 설계했다.

```csharp
public void HandleDeath()
{
    // 시스템 상의 모든 잔존 코루틴(무적 타이머 등) 중단
    StopAllCoroutines();
    
    // 사망 상태로 전환
    StateMachine.ChangeState(DeadState);
}
```
`StopAllCoroutines()` 호출을 통해 사망한 객체가 더 이상 스스로의 상태를 변경하지 못하도록 물리적으로 차단했다. 이는 '예측 가능한 상태 관리'를 위한 필수적인 안전장치이다.

## 3. Result: 견고해진 시스템

이번 개선 작업은 화려한 기능을 추가한 것은 아니나, 게임의 안정성과 사용자 경험을 한 단계 높이는 중요한 단계였다.

1.  **물리적 일관성**: 어떠한 상태에서도 기본 물리 법칙(중력)이 깨지지 않음을 보장한다.
2.  **예외 상황 차단**: 상태 전환 시 잔존하는 비동기 로직을 명시적으로 정리함으로써, 복잡한 로직 간의 충돌 가능성을 원천 봉쇄했다.
