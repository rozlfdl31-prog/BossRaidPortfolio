# [Portfolio] Boss Raid Portfolio: 2월 20일 버그 수정 회고 (AoE/페이즈/선택기)

## 1. 개요

이번 버그 수정의 목적은 "기능이 있는 상태"에서 "전투가 안정적으로 읽히는 상태"로 품질을 끌어올리는 것이었다.  
증상은 서로 달랐지만, 공통 원인은 전환 타이밍과 선택기 규칙의 불명확성에 있었다.

---

## 2. 버그 리스트와 수정 결과

| 증상 | 원인 | 수정 |
| --- | --- | --- |
| AoE 장판이 뜨거나 파묻혀 보임 | 바닥 밀착 보정값과 지면 기준 오차 | `AoECircleController`의 `fallbackYOffset=0` 적용 |
| 페이즈 인트로 중 공격이 먼저 실행됨 | 인트로 재생과 공격 선택의 타이밍 경쟁 | 전환 구간 공격 선택 잠금 적용 |
| 동일 패턴이 연속 반복됨 | 2패턴 활성 시 재선택 제한 부재 | no-immediate-repeat 규칙 추가 |
| 사거리 강제 AoE로 패턴 편중 발생 | 선택 경로 분기 과다 | 원거리 강제 분기 제거 + 공통 선택기 통합 |

---

## 3. 디버깅에서 얻은 포인트

### 3.1. "기능 동작"과 "전투 읽힘"은 다르다

패턴이 실행된다는 사실만으로는 충분하지 않았다.  
플레이어가 납득하는 타이밍과 시각 기준이 맞아야 실제 품질이 올라간다.

### 3.2. 분기보다 규칙이 강하다

사거리/상황별 분기를 계속 추가하면 단기적으로는 문제를 막아도, 장기적으로는 편중과 예외가 늘어난다.  
선택기 단일화 + 반복 방지 규칙이 유지보수 관점에서 더 안정적이었다.

## 4. Technical Insight

| 기술 포인트 | 적용 내용 | 기대 효과 |

| **Visual Ground Truth** | 장판 높이 기준 고정 | 플레이어 피드백 신뢰도 향상 |

이번 버그 수정은 개별 증상을 때우는 패치가 아니라, 보스 전투의 "규칙"을 명확히 만든 정리 작업이었다.  
다음 단계는 선택기 정책(가중치/쿨다운/최근 이력)을 데이터화해, 페이즈별 난이도 곡선을 더 정교하게 제어하는 것이다.
