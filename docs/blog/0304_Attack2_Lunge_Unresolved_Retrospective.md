## 3. 이번 코드에 남긴 것
이번 사이클은 **최종 유지할 신규 로직을 남기지 않는다**.  
현재 브랜치의 실험 코드도 기준선 확정 전에 롤백할 예정이다.

정리 대상(롤백 예정):
- `ApplyLungeRootMotion(..., normalizedTime)` 확장 경로
- `LungeAttackSettings`의 `midBoost*`/`lateReduce*` 분포 보정값
- RootMotion `scale` 추적 로그

## 4. 왜 못 고쳤는가

정확한 단일 원인은 아직 미확정이고, 현재 결론은 소거법이다.  
아래는 시도 순서(오름차순) 기준으로 정리한 소거 목록이다.

| 소거한 가설 | 어떤 시도로 소거했는가 | 관측 결과 |
| --- | --- | --- |
| 입력 소스 단일화면 해결된다 | 03-03 시도 #4 (`deltaPosition` 단일화) | 소스 단일화만 되고 기준점 오차는 유지 |
| 마커/전이 안정화만 하면 해결된다 | 03-03 시도 #5 (Launch 미진입 방지) | 미진입은 줄었지만 체감 이슈 잔존 |
| 이동량 배수 튜닝으로 해결된다 | 03-03 시도 #6 (`launch=0.6`, XZ 스케일) | 구간별 이동량 제어는 가능했지만 실플레이에서 `앞멈춤->뒤당김` 증가 |
| 토글 조합만 맞추면 해결된다 | 03-04 시도 #1 (토글 3종 분리) | 조합별 편차만 있고 동기화 일관 개선 없음 |
| Anchor 기준점만 바꾸면 해결된다 | 03-04 시도 #2 (Anchor 동기화) | red 점프/boss 정지/0이동 등 비정상 케이스 발생 |
| Soft Lock 보정이 오차를 줄인다 | 03-04 시도 #3 (Soft Lock 1차) | `used=False`, `corrXZ=0` 반복 |
| Hard Lock은 step 값 튜닝 문제다 | 03-04 시도 #6 (ON/OFF 3회) | ON에서 측면 이탈, OFF에서 전방 도약 정상 (`maxStep` 원인 아님) |
| 이동량 분포 보정으로 해결된다 | 03-04 시도 #9 (`midBoost/lateReduce`) | `scale` 반영은 확인됐지만 Exit 오차 축과 동기화 문제는 잔존 |
