# [Portfolio] Boss Raid Portfolio: 다른 환경에서 문제없이 세팅하기

## 1. 개요

이번 정리는 "내 PC에서는 되는데, 다른 PC에서 깨지는 문제"를 줄이기 위해 실제로 적용한 세팅 복구 방식이다.  
핵심은 두 가지였다.

- 대용량 외부 에셋은 각자 수동 임포트
- 프로젝트는 코드/씬/설정 + 자동 검증 도구로 동기화

---

## 2. 문제와 해결

1. 문제: 다른 PC에서 외부 에셋 미설치로 씬/프리팹 참조가 깨졌다.  
해결법: `Assets/FourEvilDragonsPBR`, `Assets/CombatGirlsCharacterPack`, `Assets/UNI VFX`는 각 PC에서 수동 임포트하도록 기준을 정했다.

2. 문제: 에셋 전체를 Git에 올리면 저장소 용량이 급증해서 free tier 한도를 압박했다.  
해결법: 벤더 에셋 원본은 커밋하지 않고, 코드/씬/프로젝트 설정만 추적하는 방식으로 운영했다.

3. 문제: "자동 다운로드로 다른 PC까지 완전 자동 세팅" 요구가 있었지만, 스토어 계정/라이선스 제약으로 일반 Git 워크플로에서 자동화가 어렵다.  
해결법: 다운로드는 수동으로 유지하고, 임포트 후 검증/보정은 에디터 도구로 자동화했다.

4. 문제: Unity는 GUID 기반 참조라 `.meta` 관리가 어긋나면 다른 PC에서 연결이 끊긴다.  
해결법: 프로젝트에서 직접 수정한 파일은 `.meta` 포함 추적, 외부 패키지는 원본 경로/구조 유지 원칙으로 맞췄다.

5. 문제: 플레이어 Animator에서 상태/모션 누락이 발생했다.  
해결법: Player Animator 필수 상태/모션 누락을 검사하는 `PlayerAnimatorGuard`를 추가했다.

6. 문제: 플레이어 공격이 보스에 데미지를 못 넣는 현상이 있었다.  
해결법: `Attack1/2/3` 클립의 `OnHitStart/OnHitEnd` 이벤트 누락을 확인했고, 이벤트 자동 보정 + 검증 로직(메뉴: `Tools/Validation/Fix Player Attack Events`)을 추가했다.

7. 문제: 보스 Animator의 Locomotion 구성에서 Idle 연결이 빠져 있었다.  
해결법: Locomotion-Idle 연결을 복구했다.

8. 문제: Player 히트 애니메이션 상수 제거 시 상태 연동이 깨질 수 있었다.  
해결법: `PlayerController`의 `ANIM_STATE_HIT = "Hit"` 상수를 유지/복구했다.

9. 문제: 점프 기능 요구사항이 혼선이었다.  
해결법: 현재 디자인 기준으로 점프는 비활성(주석 유지, F10 배치) 상태로 두고, 추후 확장 가능하게 남겼다.

---

## 3. 운영 규칙(재발 방지)

- 외부 스토어 에셋: 수동 임포트
- 프로젝트 로직/연결: 코드 + 검증 도구로 자동 점검
- 커밋 기준: 필요한 변경만 선별 커밋 (대용량 벤더 폴더 전체 커밋 금지)
- 참조 안정성: GUID/.meta 깨지지 않게 경로/구조를 일관되게 유지

---

## 4. 결론

이번 정리는 "에셋 배포"와 "프로젝트 동작 보장"을 분리한 것이 핵심이었다.  
에셋은 각 PC에서 받되, 게임 동작은 코드/씬/검증 도구로 일치시키는 방식으로 팀 환경 안정성을 확보했다.
