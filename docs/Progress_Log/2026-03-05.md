## Planned Feature Note (Player Knockback + Stun)

### Goal
Add player knockback and stun reaction when hit by boss attacks.  
Use `Assets/CombatGirlsCharacterPack/School_Katana_Girl/Animations/Normal/Stun.fbx` as stun animation.

### Inspector Fields
1. `Stun Duration`
2. `Post-Stun Invul Duration` (default: `2.0`)
3. `Pushback Duration` (default: `0.7`)
4. `Blink Frequency` (default: `0.3`)
5. `Projectile Count Timer` (default: `0.5`)

### Hit Rules (easy English)
1. Normal hit:
   - `attack1`
   - first projectile in `attack3` / `attack4` while timer is active
2. Stun hit (stun only, no damage):
   - `attack2` / `lunge`
   - second projectile in `attack3` / `attack4` inside projectile count timer

### Core Behavior (easy English)
1. If player is stunned or post-stun invulnerable, ignore new hit.
2. Stun blocks move/attack/skill/jump/interact/rotate.
3. Dash cannot cancel stun.
4. Stun interrupts current animation and plays `Stun.fbx`.
5. Pushback direction is opposite of hit direction.
6. Pushback distance = `dashSpeed * pushbackDuration`.
7. During stun: no damage.
8. After stun: post-stun invulnerability starts.
9. During post-stun invulnerability: white blink (body + weapon).
10. Blink ends when post-stun invulnerability ends.

### Implementation Result
1. Added boss hit metadata pipeline:
   - `BossAttackHitType`, `BossAttackHitData`, `IBossAttackHitReceiver`
2. Added player stun routing in `PlayerController`:
   - Attack1 -> normal damage + normal hit
   - Attack2/Lunge -> stun only
   - Attack3/4 projectile -> first hit normal, second hit stun
3. Added stun runtime control:
   - input block via `StunState`
   - pushback with duration-based distance
4. Added post-stun invulnerability + blink:
   - no damage / no re-stun in invul window
5. Added attack source classification:
   - Basic `DamageCaster` -> `Attack1`
   - Lunge `DamageCaster` -> `Attack2`
   - Projectile pattern -> `Attack3Projectile`
   - AoE tick -> `Attack4Projectile`

### 오늘 반영한 작업
* 플레이어 전용 blink 코드를 `PlayerVisual`에서 제거했다.
* 공용 컴포넌트 `BlinkWhiteEffect`를 도입했다.

## 보스 피격 받는 중 공격 시전 유지
### 오늘 반영한 작업
* White shader fix(`_BlinkWhite`)와 보스 피격 로직 정리를 같은 작업 단위로 적용했다.
* 보스가 공격 시전 중 피격되어도 공격 시전 상태는 유지되도록 처리했다.

### Core Logic (easy English)
1. If boss is in attack-casting state, hit does not force interrupt animation/state.
2. Hit still applies damage flow and white blink feedback.
3. Result: cast continuity is kept, and hit feedback is still visible.
