## 오늘 반영한 작업
* `TitleSceneController`를 `press any key` 시작점에서 버튼 기반 타이틀 메뉴 컨트롤러로 교체했다.
* `TitleMainPanel`, `MultiplayerModePanel`, `HostCreatePanel`, `ClientJoinPanel`, `LobbyPanel`, `WrongKeyPopup` 흐름을 런타임 UI로 구성했다.
* `Solo Play`는 기존 `SceneLoader -> LoadingScene -> GamePlayScene` 흐름을 유지하고, `Multi Play`는 Host/Client/Lobby 화면 전환을 제공하도록 정리했다.
* Host 경로에 optional room title, auto title(`join here 0000`), random join code, `2/2 connected` 이후 2초 `Start` unlock gate를 UI 프로토타입으로 반영했다.
* Client 경로에 6자리 join code 형식 검증과 `Wrong key. Please type again.` popup 흐름을 반영했다.
* `TitleSceneController`를 `ExecuteAlways` 기반으로 확장해 `TitleRuntimeRoot`를 Edit Mode에서도 생성/재사용하도록 바꿨다.
* 기존 `TitleRuntimeRoot`가 씬에 있으면 새로 덮어쓰지 않고 참조를 다시 묶어, Unity Editor에서 수동으로 옮긴 앵커/배치 수정이 유지되도록 정리했다.
* `Progress_Log`, `System_Blueprint`, `Technical_Glossary`를 현재 구현 기준으로 동기화했다.

## 기술적 고려
### 무엇을 발견했는가
* 기존 `TitleSceneController`는 `Input.anyKeyDown`만 감지하는 단일 진입점 구조라서 `Solo / Multi / Host / Client / Lobby` 패널을 담을 수 없었다.
* 현재 `TitleScene`에는 배경과 `Text_PressAnyKey`만 있고, 멀티플레이 UI 오브젝트/패널 계층이 전혀 없는 상태였다.
* 이 단계에서 씬 YAML을 크게 손대면 UI 프리팹/오브젝트 참조까지 한 번에 늘어나 회귀 범위가 커질 가능성이 있었다.
* 플레이 모드에서 생성된 `TitleRuntimeRoot`는 정지 시 사라져서, Host/Client/Lobby 패널의 위치와 크기를 Editor에서 직접 다듬을 수 없었다.

### 무엇을 수정했는가
* `TitleSceneController`가 기존 캔버스를 찾아 런타임에 타이틀 카드, 버튼, 입력 필드, 로비 표시 영역, wrong key popup을 생성하도록 변경했다.
* `Solo Play` 버튼은 기존 전투 진입 흐름을 그대로 호출하고, `Multi Play`는 `Host`/`Client` 분기 패널을 여는 방식으로 바꿨다.
* Host는 room title이 비어 있으면 `join here 0000` 형식의 자동 제목을 생성하고, random join code와 `Start` unlock timer를 표시하도록 했다.
* Client는 6자리 영숫자 join code만 허용하고, 형식이 틀리면 popup을 띄운 뒤 입력 패널에 머무르게 했다.
* `TitleSceneController`에 `ExecuteAlways`와 Edit Mode 가드를 추가해, 플레이 중이 아닐 때도 `TitleRuntimeRoot`를 만들고 씬에 유지하도록 바꿨다.
* 이미 존재하는 `TitleRuntimeRoot`는 새로 생성하지 않고 내부 버튼/입력 필드 참조와 클릭 이벤트만 다시 연결하도록 분리했다.
* `SceneLoader` 호출과 카운트다운 갱신은 Play Mode에서만 실행되도록 분기해 Edit Mode에서 런타임 흐름이 섞이지 않게 했다.

### 왜 그렇게 판단했는가
* 문서 우선 워크플로에서 첫 단계는 서비스 연동보다 UI 구조 고정이므로, 네트워크 구현보다 패널 계약과 전환 규칙을 먼저 눈에 보이게 만드는 편이 안전하다.
* 기존 씬 구조를 크게 변경하지 않고 런타임 UI를 올리면 현재 `TitleScene` 배경과 씬 참조를 유지하면서도 회귀 범위를 최소화할 수 있다.
* 실제 `Relay / Lobby / NGO`를 붙이기 전에 버튼, 입력, popup, lobby 상태 표시가 먼저 정리되어야 다음 단계에서 서비스 레이어 연결이 단순해진다.
* Title UI를 씬에 남겨 두고 재사용해야 디자이너가 Play Mode 진입 없이 패널 배치와 앵커를 바로 조정할 수 있고, 실행 시에도 같은 오브젝트를 그대로 써서 편집 결과가 보존된다.
