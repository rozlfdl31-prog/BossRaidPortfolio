## Attack2/Lunge 재정리 (사용자 요청 반영: 시도/결과/실패 원인)
* 아래 항목은 2026-03-03에 실제로 시도한 변경을 "무엇을 했고, 어떤 결과였고, 왜 실패했는지" 기준으로 재정리한 것이다.
* 1) Attack2를 `Windup/PreLaunch/Launch/Airborne/Land`로 분리 제어
  * 결과: 전이 지점 관측과 디버깅은 쉬워졌고, 전이 누락 추적은 개선됐다.
  * 실패 원인: 전이 가시성 개선과 실제 동기화/슬라이드 해결은 다른 문제였고, 본체 이동 경로 불일치가 남아 체감 문제가 지속됐다.
* 2) 도약 직전 지면 잠금 + `stepOffset=0` + Y 처리 + 중력 억제 토글
  * 결과: 특정 구간의 접지 안정성은 일부 개선됐다.
  * 실패 원인: 지면 잠금/중력/루트모션이 함께 개입해 이동 권한이 분산되었고, 동기화 목표(보스-레드 동시 이동)에는 직접 해법이 되지 못했다.
* 3) 방향 고정/재투영 보정 제거 + 루트모션 델타 직결
  * 결과: 경로는 단순해졌지만 슬라이드/당김 체감이 다시 나타났다.
  * 실패 원인: 단순화 이후에도 입력 델타 자체의 기준점 불일치(보스 본체 vs 비주얼/레드)가 해소되지 않아 결과 불일치가 유지됐다.
* 4) `visualDelta fallback` 제거 후 `animator.deltaPosition` 단일 소스 사용
  * 결과: 델타 소스는 단일화됐으나, 기대한 동기화 개선은 확인되지 않았다.
  * 실패 원인: `animator.deltaPosition` 단일 소스만으로는 보스-레드 기준점 오차를 보정하지 못했고, 일부 구간에서는 이동 입력 품질이 오히려 취약해졌다.
* 5) Launch 미진입 방지(`0.98` 상한 + synthetic marker)
  * 결과: Launch 영구 미진입 경로는 차단됐다.
  * 실패 원인: 전이 안정화는 되었지만, 핵심 체감 이슈(슬라이드/당김/비동기)는 별도 축이라 근본 해결이 아니었다.
* 6) Launch 상한 `0.6` + XZ 스케일(`ground 0.1 / airborne 1.15`) 튜닝
  * 결과: 구간별 이동량 제어는 가능했다.
  * 실패 원인: 실플레이 체감에서 "앞에서 멈춤 -> 뒤로 당김"이 커져 회귀로 판단했고, 해당 튜닝은 롤백됐다.
* 7) 슬라이드 재발 후 스냅샷 저장 + 런타임 3파일 롤백
  * 결과: 비교 가능한 기준선은 회복됐다.
  * 실패 원인: 문제를 해결한 것이 아니라 실험 누적으로 흐려진 기준을 복구한 단계였다.
